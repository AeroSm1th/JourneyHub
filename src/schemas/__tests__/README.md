# Schema 单元测试文档

## 概述

本目录包含所有 Zod schema 的单元测试，确保表单验证规则的正确性和完整性。

## 测试文件

**测试覆盖：**

- ✅ 国家名称验证（必填、长度限制）
- ✅ 大洲枚举验证
- ✅ 坐标范围验证（纬度 -90~90，经度 -180~180）
- ✅ 访问日期验证（不能是未来日期）
- ✅ 旅行类型枚举验证
- ✅ 评分验证（1-5 整数）
- ✅ 备注长度验证（最多 2000 字符）

- ✅ 图片上传验证（格式、大小限制）
- ✅ 愿望清单优先级验证（1-5）
- ✅ 期望季节枚举验证

## 运行测试

```bash
# 运行所有 schema 测试
npm run test:run -- src/schemas/__tests__

# 运行特定测试文件
npm run test:run -- src/schemas/__tests__/citySchema.test.ts

# 运行测试并生成覆盖率报告
npm run test:coverage -- src/schemas/__tests__
```

## 边界条件测试

所有测试都包含了边界条件的验证：

1. **最小值/最大值边界**
   - 字符串长度的最小和最大值
   - 数字范围的最小和最大值
   - 数组长度的限制

2. **空值处理**
   - 空字符串
   - undefined
   - null

3. **类型错误**
   - 字符串传入数字字段
   - 数字传入字符串字段
   - 无效的枚举值

4. **格式验证**
   - 邮箱格式
   - UUID 格式
   - 日期格式
   - 文件类型和大小

## 错误消息验证

每个测试都验证了错误消息的正确性，确保用户能够获得清晰的反馈。

**注意：** Zod v4 的某些内置错误消息格式与 v3 不同，测试中使用 `toContain()` 来验证关键错误信息。

## 维护指南

### 添加新的验证规则

1. 在对应的 schema 文件中添加新的验证规则
2. 在对应的测试文件中添加测试用例
3. 确保测试覆盖正常情况和边界条件
4. 运行测试确保通过

### 修改现有验证规则

1. 更新 schema 文件中的验证规则
2. 更新对应的测试用例
3. 运行测试确保所有测试通过
4. 更新本文档中的测试覆盖说明
